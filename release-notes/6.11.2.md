## <a name="bugs"></a>Bugs resolved

* **_Composer_: Update package "firebase/php-jwt" ([#34817](https://github.com/civicrm/civicrm-core/pull/34817))**

  php-jwt v7.0 is nominally a security update.  The security angle does not appear to affect normal CiviCRM configurations. Never-the-less, auditing tools may make noise about it. This update suppresses the noise.

* **_Contact Summary_: Field "Preferred Language" does not save ([#34882](https://github.com/civicrm/civicrm-core/pull/34882))**
* **_Display Preferences_: Restore missing option on administration screen ([dev/core#6351](https://lab.civicrm.org/dev/core/-/issues/6351): [#34877](https://github.com/civicrm/civicrm-core/pull/34877))**
* **_Form Builder_: Field overrides are lost in translation ([dev/core#6207](https://lab.civicrm.org/dev/core/-/issues/6207): [#34892](https://github.com/civicrm/civicrm-core/pull/34892))**
* **_Form Builder_: Does not open in PSR-16 "strict" mode ([dev/core#6353](https://lab.civicrm.org/dev/core/-/issues/6353): [#34887](https://github.com/civicrm/civicrm-core/pull/34887))**
* **_Riverlea_: Front-end screens display extra padding on "toggle" widgets ([#34880](https://github.com/civicrm/civicrm-core/pull/34880))**
* **_Riverlea_: Fix renewal checkbox when using price-sets for memberships ([#34952](https://github.com/civicrm/civicrm-core/pull/34952))**
* **_Riverlea_: Fix highlighting on certain dropdowns ([#34940](https://github.com/civicrm/civicrm-core/pull/34940))**
